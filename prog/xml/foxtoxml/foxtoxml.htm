FoxToXML Sample

About This Sample Program

FoxToXML.prg is a program file that creates a custom Microsoft Visual FoxPro (VFP) class that uses a VFP cursor or table to create an eXtensible Markup Language (XML) page and an eXtensible Stylesheet Language (XSL) style sheet to display the data.

The VFP class (FoxToXML) has a method that will create an XML file, a method that will create an XSL file, and a method that will create both files. Each method either:
  • Takes no parameter and uses the currently open cursor or table to create the XML/XSL files.

    -or-

  • Takes a table name as a parameter to create the files from the specified table.
The creation of the XML/XSL documents uses native Visual FoxPro 6.0 language and the custom class can be instantiated and used whenever an XML or XSL document is needed. The XML/XSL document names are derived from the cursor or table name.

Without the XSL file, the XML data will display in hierarchial form. To see how the data will look without the style sheet, create an XML file and delete the "?xml-stylesheet ..." line in the XML file.

Using the FoxToXML Class

  1. Create a new program called Test.prg and add the following code:

    CD JUSTPATH(SYS(16))
    USE ? && Choose any FoxPro table
    oXML = NEWOBJECT("FoxToXML","FoxToXML.prg")
    oXML.BOTH
    ** This will create both an XML and XSL file with the name of the
    ** open table or cursor.
  2. Save the program in the same directory as the FoxToXML.prg file.

  3. Run Test.prg.

How It Works

The class contains the following methods:
  • CheckAlias - This method checks to see whether there is a data source. If there is no data source, no XML or XSL file is created.
  • UseThisData - This method accepts a table name as a parameter. If the table physically exists and is not currently being used by the current instance of Visual FoxPro, then this method will open the table. If the table is currently open, the table will be selected.
  • pTab - This method inserts tab characters into the output files. This is done strictly for readability.
  • XSL - This method creates the XSL file and accepts a table name as a parameter. If no table is specified but a table or cursor is open, this method uses that data structure to create the style sheet. The name of the resultant XSL file will be the table or cursor name (for example, MyTable.xsl).
  • XML - This method creates the XML file and accepts a table name as a parameter. If no table is specified but a table or cursor is open, this method uses that data to create the XML file. The name of the resultant XML file will be the table or cursor name (for example, MyTable.xml).
  • BOTH - This method runs the XML and XSL methods described above.
